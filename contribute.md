# Contributing to VulnVault

Thanks for your interest in contributing to **VulnVault** 

This project exists to help developers build secure software through **clear, practical, and accurate guidance**. Contributions that improve clarity, correctness, or coverage are always welcome.

---

## What You Can Contribute

You can contribute by:

- Adding new vulnerabilities
- Adding new languages or frameworks
- Improving insecure / secure code examples
- Fixing inaccuracies or edge cases
- Improving documentation and explanations
- Aligning content with new standards or best practices

## Structure Guidelines

### Vulnerability Content

Each vulnerability should include:

1. Description & Risk
2. How the Vulnerability Occurs
3. Insecure Example(s)
4. Secure Example(s)
5. Prevention & Best Practices
6. Relevant Standards (OWASP Top 10, ASVS, CWE)

### Code Examples

- Clearly label insecure vs secure code
- Include comments where helpful
- Avoid external dependencies unless required

---

## Pull Request Process

1. Fork the repository
2. Create a feature branch
3. Make your changes
4. Ensure examples are accurate and build logically
5. Submit a pull request with a clear description

Please include:
- What vulnerability or improvement is being addressed
- Languages/frameworks impacted
- Any relevant standards references

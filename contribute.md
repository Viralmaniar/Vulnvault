## Structure Guidelines

### Vulnerability Content

Each vulnerability should include:

1. Description & Risk
2. How the Vulnerability Occurs
3. Insecure Example(s)
4. Secure Example(s)
5. Prevention & Best Practices
6. Relevant Standards (OWASP Top 10, ASVS, CWE)

### Code Examples

- Clearly label insecure vs secure code
- Include comments where helpful
- Avoid external dependencies unless required

---

## Pull Request Process

1. Fork the repository
2. Create a feature branch
3. Make your changes
4. Ensure examples are accurate and build logically
5. Submit a pull request with a clear description

Please include:
- What vulnerability or improvement is being addressed
- Languages/frameworks impacted
- Any relevant standards references
